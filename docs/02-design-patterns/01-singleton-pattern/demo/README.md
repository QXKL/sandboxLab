# 单例模式 - 代码示例

## 示例说明

本示例展示了单例模式的6种实现方式及其对比：

1. **饿汉式**（线程安全，类加载时创建）
2. **懒汉式**（线程不安全）
3. **懒汉式同步方法**（线程安全，性能差）
4. **双重检查锁定DCL**（线程安全，性能好）
5. **静态内部类**（推荐，线程安全+延迟加载）
6. **枚举单例**（最安全，防反射和序列化攻击）

## 核心要点

### 1. 单例模式的三要素
- 私有构造函数
- 私有静态实例
- 公共静态访问方法

### 2. 线程安全问题
- 懒汉式在多线程下不安全
- 需要同步机制或利用类加载机制

### 3. DCL为什么需要volatile？
- 防止指令重排序
- 保证可见性

### 4. 静态内部类的优势
- 延迟加载（只在调用时加载）
- 线程安全（类加载机制保证）
- 实现简单，无需同步

### 5. 枚举单例的优势
- 防止反射攻击
- 防止序列化创建新实例
- 代码最简洁

## 最佳实践

**推荐使用**：
1. **静态内部类**（通用场景）
2. **枚举**（需要防攻击）
3. **饿汉式**（不需要延迟加载）

**避免使用**：
- 懒汉式（不同步）- 线程不安全
- 懒汉式（同步方法）- 性能差

## 记住

> **静态内部类最优雅，枚举方式最安全！**
