## 如何判断是否符合 OCP？

### 判断方法1: 扩展测试
```
问自己：如果要新增一个功能，需要修改多少现有代码？

符合OCP: 只需添加新类，不修改旧代码
违反OCP: 需要修改原有类的代码
```

### 判断方法2: if-else/switch-case检查
```
如果代码中有大量的 if-else 或 switch-case 判断类型，
很可能违反了OCP。

解决方法：[用什么方式替代？]
```

### 判断方法3: 依赖检查
```

符合OCP: 依赖接口
违反OCP: 依赖具体类
```
